欢迎来到 **OPD Learning Lab**。这里的目标不是让你背下一个 loss，而是让你能回答四个问题：

1. 为什么训练数据必须来自学生当前会到达的状态？
2. 论文里的 KL，最终在代码里变成了哪些 tensor 和 mask？
3. teacher、rollout、actor 在 tinker 与 verl 中如何交接数据？
4. 训练曲线变好时，怎样判断模型真的学到了，而不是熵塌缩、长度漂移或评测偶然波动？

## 整门课的主线

```text
prompt x
  │
  ├─ student rollout:  ŷ ~ πθ(. | x)          ← 决定“在哪里学”
  │
  ├─ teacher scores:   log πT(ŷt | x, ŷ<t)    ← 决定“老师怎么说”
  │
  ├─ student scores:   log πθ(ŷt | x, ŷ<t)
  │
  ├─ estimator + mask + aggregation            ← 决定“代码实际优化什么”
  │
  └─ update θ, refresh rollout policy           ← 决定“下一轮数据来自谁”
```

你会反复看到四层概念。请始终把它们分开：

- **状态分布**：prompt 与 response prefix 从谁来；
- **目标散度**：想让哪两个分布接近；
- **估计器**：full vocabulary、top-k 还是 sampled token；
- **系统实现**：teacher 怎样服务、token 怎样对齐、权重何时刷新。
