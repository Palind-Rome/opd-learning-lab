一口气打开几十篇 OPD 论文，最容易得到的不是理解，而是一串互相覆盖的缩写。更有效的顺序是：**先建立状态分布直觉，再读一个最小实现，然后补公式与失败模式，最后才追前沿变体。**

<Callout title="这份清单只放一手入口">
  论文链接指向 arXiv，代码指向作者或机构仓库，工业实践指向技术报告或官方项目页。Awesome 列表适合发现条目，但不替代原论文，也不构成质量背书。
</Callout>

## 第 0 层：只补真正需要的前置知识

不必先学完整套强化学习，但下面四个概念必须会说人话：

### 0.1 自回归概率

理解一句话的概率如何分解为每个 token 的条件概率，以及 teacher forcing 为什么让训练前缀与推理前缀不同。OPD 的出发点正是：学生需要在**自己会访问的前缀**上学习恢复。

### 0.2 FKL、RKL 与 JSD

- Forward KL 常被解释为 mode-covering：遗漏教师支持的模式代价大；
- Reverse KL 常被解释为 mode-seeking：学生更倾向集中到教师高概率模式；
- JSD 用混合分布缓和两个方向。

这只是直觉。实现中是全词表求和、top-k 截断还是单 token 采样，往往比“FKL/RKL”标签更重要。

### 0.3 SFT、RL 与 OPD 的差别

| 方法 | 状态/轨迹来自 | 监督密度 |
|---|---|---|
| SFT / 离线蒸馏 | 固定示范或教师轨迹 | 稠密，但状态偏教师 |
| RLVR / RLHF | 当前学生 rollout | 常为序列级、较稀疏 |
| OPD | 当前学生 rollout | 教师提供 token/局部/轨迹监督 |

### 0.4 Policy gradient 与 PPO 的最低限度

阅读 [PPO 原论文](https://arxiv.org/abs/1707.06347) 的摘要、surrogate objective 和 clipping 即可。你需要理解旧策略采样、importance ratio、advantage 与多轮 minibatch 更新；不必先掌握所有连续控制实验。

## 第 2 层：补齐历史基线与目标函数

建议顺序如下：

1. [MiniLLM](https://arxiv.org/abs/2306.08543)：用 policy-gradient 视角优化 reverse KL，理解为什么学生采样能缓解 exposure bias；[代码](https://github.com/microsoft/LMOps/tree/main/minillm)。
2. [GKD](https://arxiv.org/abs/2306.13649)：比较教师轨迹、学生轨迹及其混合，理解 generalized JSD。
3. [DistiLLM](https://arxiv.org/abs/2402.03898)：看 skew KL 和 adaptive off/on-policy 数据使用；[代码](https://github.com/jongwooko/distillm)。
4. [DistiLLM-2](https://arxiv.org/abs/2503.07067)：理解教师数据和学生数据为什么可以使用不对称目标；[代码](https://github.com/jongwooko/distillm-2)。
5. [Qwen3 Technical Report](https://arxiv.org/abs/2505.09388)：看工业配方如何把 off-policy cold start 与 on-policy phase 串起来。

这一层的学习成果不是背下 KL 性质，而是能画出：**谁采样、谁打分、在哪个前缀、对哪个词表支持、梯度流向哪里。**

## 第 3 层：源码路线

### 路线 3A：最小代码

打开 [tinker-cookbook distillation 目录](https://github.com/thinking-machines-lab/tinker-cookbook/tree/main/tinker_cookbook/recipes/distillation)，依次回答：

1. rollout 请求使用哪个模型和采样配置？
2. teacher log-prob 如何与学生 token 对齐？
3. response token 的 mask 在哪里产生？
4. loss 是 full distribution 还是 sampled-token estimator？
5. teacher 是否冻结？同一 batch 会复用几次？

### 路线 3B：TRL trainer

[TRL experimental](https://github.com/huggingface/trl/tree/main/trl/experimental) 适合横向对照 GKD、GOLD、MiniLLM、SDFT 和 self-distillation。重点看 trainer 如何覆写：

- generation / data collation；
- teacher forward；
- KL/JSD 计算；
- Accelerate/DeepSpeed 下的模型放置。

experimental API 可能变化，阅读时记录 commit，而不是只写“使用最新版 TRL”。

### 路线 3C：verl 分布式数据流

先读 [verl OPD 文档](https://verl.readthedocs.io/en/latest/algo/opd.html)，再进入 [recipe/on_policy_distill](https://github.com/volcengine/verl/tree/main/recipe/on_policy_distill)。把系统拆成：

`prompt → actor rollout → teacher/ref scoring → token advantage/loss → actor update → weight sync`

然后检查：

- actor、rollout 和 ref 是否同卡；
- teacher 返回 full logits、top-k 还是 sampled log-prob；
- old log-prob 与 current log-prob 在哪里计算；
- response mask、tool mask 与 padding mask 是否分开；
- 异步模式下 rollout 相对当前 actor 陈旧多少步。

## 第 4 层：失败模式是必读，不是“进阶选修”

按下面顺序读，能少做很多无效超参搜索。

1. [Rethinking OPD](https://arxiv.org/abs/2604.13016)：教师与学生的 thinking pattern、教师是否真的有新能力、cold start 与 prompt selection。
2. [Revisiting OPD](https://arxiv.org/abs/2603.25562v2)：单 token 信号、学生前缀上的教师可靠性、tokenizer/特殊 token。
3. [The Many Faces of OPD](https://arxiv.org/abs/2605.11182)：分布失配、biased top-k、初始化与 OPSD 的 PI-free policy 限制。
4. [Veto](https://arxiv.org/abs/2601.07155)：学生–教师间隙太大时的目标重构。
5. [Entropy-Aware OPD](https://arxiv.org/abs/2603.07079)：高熵教师 token 与生成多样性。

<Callout title="失败论文之间并不矛盾">
  它们分别控制 teacher 选择、学生状态、词表支持、目标几何、初始化与 privileged information。一个设置可能同时踩中多种故障。先定位故障轴，再选择修复，比直接堆叠所有新 loss 更可靠。
</Callout>

## 第 5 层：按研究方向进入 2026 前沿

### 长链推理与信用分配

1. [IW-OPD / On the Position Bias of OPD](https://arxiv.org/abs/2606.22600)
2. [Prune-OPD](https://arxiv.org/abs/2605.07804)
3. [Trajectory-Refined Distillation](https://arxiv.org/abs/2606.08432) / [代码](https://github.com/louieworth/trd)

比较时分别记录：position weight、drift detector、trajectory rewrite、hard filter/soft weight、trust-region outlier handling。不要把它们统称为“只训练重要 token”。

### OPSD 与上下文内化

1. [Self-Distilled Reasoner](https://arxiv.org/abs/2601.18734) / [代码](https://github.com/siyan-zhao/OPSD)

### 黑盒、Agent 与多模态

- 黑盒：[GAD](https://ytianzhu.github.io/Generative-Adversarial-Distillation/)、[OVD](https://arxiv.org/abs/2601.21968)、[ROPD](https://github.com/Peregrine123/ROPD_official)
- Agent：[TCOD](https://arxiv.org/abs/2604.24005)、[Healthcare AI GYM](https://github.com/minstar/Healthcare_GYM)、[SDAR](https://github.com/ZJU-REAL/SDAR)

## 读任何新论文时，用这张八问卡

1. **状态是谁产生的？** 当前学生、旧学生、教师，还是混合策略？
2. **教师是谁？** 外部模型、同模型 + PI、旧 checkpoint、多教师、verifier 还是 rubric？
3. **教师能提供什么？** full logits、top-k、sampled log-prob、hidden state、文本反馈或标量？
4. **优化对象是什么？** FKL、RKL、JSD、policy gradient、MLE，还是它们的路由组合？
5. **实际 estimator 是什么？** 全词表、截断支持、单 token、return-to-go、是否 stop-gradient？
6. **哪些数据被过滤或加权？** token、位置、segment、turn、trajectory、正确性、PPL？
7. **证据有多强？** 哪些模型、数据、预算、指标和消融支持结论？
8. **代码真的公开到什么程度？** 论文仓库、训练入口、配置、数据、评测、commit 是否齐全？

能回答这八问，你就已经不是在“收集缩写”，而是在比较训练算法。
